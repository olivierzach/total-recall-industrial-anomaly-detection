Hybrid Memory Retrieval

Open-Set Failure Detection With Labeled Memory

This interface combines a nominal anomaly detector with a labeled failure memory bank. The system first estimates whether an image is off-nominal, then decides whether the anomaly is close to a previously stored failure mode or should remain categorized as a novel anomaly.

Metric Glossary
Retrieved Label

The retrieved known failure class or support family. It is only trusted when the image is anomalous and the failure descriptor is sufficiently close to one stored class.

Anomaly Score

The PatchCore image-level distance from the nominal memory bank. Larger values mean the image looks less like normal training data.

Threshold

The anomaly cutoff calibrated on held-out nominal images. Scores below it are treated as normal; scores above it move to known-vs-unknown failure retrieval.

Margin

The ratio between the second-best and best known-failure distances. Higher margin means the top retrieved failure class is more clearly separated from alternatives.

Embedding Heatmap

A per-patch embedding-space response relative to the nearest retrieved failure memory. It is a feature-response view, not a calibrated anomaly probability map.
